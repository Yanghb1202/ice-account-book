## 二、本地部署步骤

### 环境要求
- 微信开发者工具（版本 1.05 以上）
- Node.js 14+（用于云函数开发）
- 微信小程序账号（需开通云开发）

### 部署步骤

#### 1. 下载源码
```bash
git clone https://github.com/Yanghb1202/ice-account-book.git
cd ice-account-book
```

#### 2. 配置云开发环境

**步骤1：开通云开发**
- 打开微信开发者工具
- 点击工具栏的"云开发"按钮
- 选择"开通云开发"，创建云环境

**步骤2：配置环境ID**
- 在云开发控制台复制环境ID
- 修改 `app.js` 文件：
```javascript
wx.cloud.init({
  env: 'your-cloud-env-id',  // 替换为你的云环境ID
  traceUser: true
})
```

**步骤3：创建数据库集合**
在云开发控制台创建以下集合：
- `bills` - 账单数据
- `users` - 用户数据

**步骤4：设置数据库权限**
- 进入云数据库控制台
- 将 `bills` 和 `users` 集合的权限设置为："所有用户可读，所有用户可写"

#### 3. 上传云函数

**步骤1：安装依赖**
```bash
cd cloudfunctions/userLogin
npm install
```
对所有云函数目录执行此操作：`addBill`, `getBillList`, `registerUser`, `foodAi`, `voiceRecognition`

**步骤2：上传云函数**
- 在微信开发者工具中右键云函数目录
- 选择"上传并部署：云端安装依赖"
- 等待部署完成（每个云函数约1-2分钟）

#### 4. 配置本地开发模式（可选）

如果暂时无法配置云开发，可以使用本地存储模式：
- 修改 `utils/db.js`：
```javascript
const useCloud = false  // false=本地存储模式
```

#### 5. 运行项目

- 在微信开发者工具中点击"编译"
- 使用微信扫码预览或在模拟器中测试

## 四、常见问题

### Q1：云函数部署失败？
**解决方案**：
- 确保Node.js版本在14+
- 检查package.json中的依赖是否完整
- 重新右键云函数目录，选择"上传并部署：云端安装依赖"

### Q2：数据库权限错误？
**解决方案**：
- 在云开发控制台设置数据库权限为"所有用户可读，所有用户可写"
- 确保云函数中使用了正确的集合名称

### Q3：图片上传失败？
**解决方案**：
- 确保云存储已开通
- 检查图片大小限制（建议小于1MB）
- 确保网络连接正常

### Q4：本地存储模式数据不持久？
**解决方案**：
- 本地存储模式数据存储在用户设备上
- 切换到云开发模式可实现数据持久化
