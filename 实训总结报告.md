# AI 辅助编程与工程化实训总结报告

## 一、项目概述

本次实训项目为"冰冰记账本"微信小程序，是一款面向个人用户的财务管理应用。项目涵盖首页仪表盘、智能记账、数据统计、预算管理、存钱挑战、餐饮AI识别等核心功能模块，采用微信小程序原生开发框架搭配云开发技术栈实现。

## 二、遇到的最大困难及解决过程

在项目开发过程中，最大的困难是**数据层架构设计**和**云开发与本地存储的兼容**问题。

### 问题背景

项目初期采用纯本地存储方案（`wx.getStorageSync`），所有数据存储在用户设备上。但考核方案要求"至少3个API接口"和"线上可访问的Demo"，这意味着必须引入云开发。然而直接将所有代码从本地存储迁移到云开发存在以下挑战：

1. **兼容性问题**：已有的27个页面都依赖本地存储，全面迁移工作量巨大
2. **网络依赖**：云开发需要网络连接，离线时无法使用
3. **用户体验**：云函数调用有延迟，需要添加loading状态和错误处理

### 解决过程

经过分析，我采取了**渐进式迁移**和**双模式兼容**的策略：

1. **创建统一的数据封装层**：在 `utils/db.js` 中实现了云开发和本地存储的双模式切换机制，通过 `useCloud` 变量控制，默认使用本地存储模式确保项目随时可运行

2. **优先迁移核心接口**：先实现了5个核心云函数（`userLogin`、`registerUser`、`addBill`、`getBillList`、`foodAi`），满足考核要求的"至少3个API接口"

3. **逐步改造页面**：选择4个核心页面（登录、注册、记账、首页）进行云开发适配，其他页面保持本地存储不变

4. **添加错误处理和降级机制**：在云函数调用失败时自动降级到本地存储模式，确保用户体验不受影响

5. **AI食物识别的降级方案**：由于外部AI API需要密钥，在 `foodAi` 云函数中实现了模拟数据返回，确保功能完整性

### 关键代码示例

```javascript
// utils/db.js - 双模式切换
const useCloud = false

const DB = {
  addBill: async (data) => {
    const userId = getUserId()
    if (useCloud && userId) {
      return await cloudFn('addBill', { ...data, userId })
    }
    return await localFn.addBill(data)
  }
}
```

## 三、对 AI 辅助编程的心得体会

通过本次实训，我深刻体会到 AI 辅助编程带来的效率提升和学习价值。

### 效率提升

1. **代码生成**：AI 能够快速生成代码模板，如计算器键盘、图表配置、云函数等，节省了大量重复劳动
2. **架构设计**：通过与 AI 讨论项目架构，获得了专业的设计建议，避免了很多弯路
3. **文档生成**：AI 帮助生成了完整的 README.md、API 文档和 Prompt 日志，提升了项目的规范性

### 学习价值

1. **快速上手新技术**：通过 AI 的代码解释和优化建议，快速掌握了微信云开发、u-charts 图表库等新技术
2. **代码质量保障**：AI 提供的代码结构清晰、注释完整，帮助我养成了良好的编码习惯
3. **问题解决能力**：遇到问题时，AI 能够提供多种解决方案，培养了我的问题分析能力

### 局限性与应对

1. **需要人工干预**：AI 生成的代码需要结合业务实际进行调整和优化，不能完全依赖
2. **Prompt 设计很重要**：清晰、详细的 Prompt 能够获得更准确的响应结果
3. **代码安全性**：需要人工审查 AI 生成的代码，确保没有安全漏洞
